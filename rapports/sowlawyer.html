Risque
⚠ 3
Niveau critique — 3 failles actives
La porte d'accès interne de votre site est publiquement visible, votre module WooCommerce n'est pas à jour, et trois protections réseau essentielles sont absentes. Ces points réunis exposent les données confidentielles de vos clients à des attaques automatisées.
Point 01
Votre porte d'accès interne est visible par n'importe qui
Accès gestion /wp-admin
Publiquement exposé
La page de connexion répond publiquement (code HTTP 200). N'importe quel robot peut la localiser et tenter de forcer votre mot de passe sans limite de tentatives.
Protection anti-robots
Absente
Aucun pare-feu, CAPTCHA ni restriction d'accès détecté. Les tentatives d'intrusion ne rencontrent aucun obstacle vers votre interface de gestion.
Pourquoi c'est critique pour un cabinet d'avocats
Un accès administrateur compromis expose l'intégralité du site — formulaires, données clients, documents partagés.
Le secret professionnel s'applique au numérique : la responsabilité du cabinet peut être engagée en cas de fuite de données issues de prises de contact en ligne.
Protection manquanteNiveau de risqueConséquence concrète
X-Frame-Options
Élevé
Un attaquant peut superposer votre site dans une fausse page pour intercepter clics ou identifiants de vos visiteurs.
X-Content-Type-Options
Moyen
Le navigateur peut être amené à exécuter des fichiers malveillants en les interprétant comme du code légitime.
Strict-Transport-Security
Élevé
Malgré votre certificat SSL actif, certains navigateurs peuvent basculer sur une connexion non chiffrée à votre insu.

À jour

WooCommerce

Version actuelle : 10.8.1

Extension historiquement ciblée. Son maintien à jour est non négociable.
Le risque combiné WooCommerce + accès admin exposé
Un module de paiement non mis à jour sur un site dont l'admin est accessible est une cible documentée pour l'injection de scripts capturant silencieusement les données de formulaires.
Les failles corrigées dans WooCommerce 10.4 à 10.8.1 restent actives sur votre installation. Les attaquants connaissent précisément ces versions vulnérables.
Point 03
Un site trop lent sur smartphone — des clients potentiels qui abandonnent
Score mobile — Google PageSpeed
28–34 / 100
L'élément principal met plus de 8,5 secondes à apparaître sur smartphone. À Dakar, avec une connexion 3G/4G, la majorité des visiteurs abandonne avant ce délai.
Score desktop — Google PageSpeed
65–71 / 100
Score moyen sur ordinateur. Images de la galerie en JPEG/PNG lourd au lieu de formats modernes compressés (WebP/AVIF).
Ce qui fonctionne bien — acquis à préserver
Certificat SSL Let's Encrypt actif — connexions entre visiteurs et site chiffrées.
Infrastructure LWS avec CDN actif — distribution du contenu accélérée en réseau.
Elementor maintenu à jour — surface d'attaque du constructeur de pages maîtrisée.
