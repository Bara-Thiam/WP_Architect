Critique
⚠ 3
CMS à jour, écosystème en retard — risque actif documenté
Votre site tourne sur WordPress 7.0, la toute dernière version majeure (mai 2026) — c'est un excellent choix. Mais l'écosystème qui l'entoure ne suit pas : une extension affectée par une CVE active, un serveur web exposé, et des performances si dégradées que nos outils de mesure n'ont pas pu terminer leur analyse.
Point 01
Un cœur WordPress à jour — un investissement technique visible
WordPress Core
7.0 — À jour
Version majeure stable sortie en mai 2026. Votre équipe (ou votre prestataire) maintient le cœur du site à jour — c'est une fondation sérieuse que peu d'agences immobilières à Dakar ont.
Elementor
4.0.8 — Récent
Le constructeur de pages est dans une version récente. La surface d'attaque du builder visuel est correctement maîtrisée.
Point 02
Une vulnérabilité active et une dette technique qui fragilisent cette bonne base
Slider Revolution CVE-2024-8107
Vulnérable
Version détectée : 6.7.18 — Version actuelle : 6.7.35
Cette version est affectée par une faille XSS documentée permettant l'injection de code malveillant via l'upload de fichiers SVG.
Serveur web nginx
1.25.5 — Exposé
Cette version s'inscrit dans la plage critique liée à CVE-2024-31079 (faille HTTP/3 de type stack overflow). Sans correctif confirmé, le service est potentiellement exploitable.
Ce que la faille Slider Revolution signifie concrètement pour une agence immobilière
Un utilisateur malveillant avec un simple compte "Auteur" peut injecter du code JavaScript dans vos pages — interceptant silencieusement les données saisies dans vos formulaires de contact et d'estimation.
Vos clients transmettent des documents sensibles dans le cadre de leurs dossiers locatifs ou d'achat (pièces d'identité, bulletins de paie). Une compromission de formulaire sur un site non patché engage directement la responsabilité de l'agence.
Le correctif est disponible depuis la version 6.7.19. La mise à jour est la seule action qui ferme cette faille — et elle doit être testée sur WordPress 7.0 avant déploiement.

Point 03
Des performances si dégradées que nos outils n'ont pas pu terminer leur analyse
Score mobile — Google PageSpeed
Timeout
La page charge si lentement sur smartphone que notre outil de mesure (Lighthouse CLI) a atteint sa limite de temps sans pouvoir finir l'analyse. Aucun score n'a pu être généré.
Score desktop — Google PageSpeed
Timeout
Même constat sur ordinateur. Le site est trop lent pour être mesuré — ce qui signifie qu'il est également trop lent pour être bien référencé par Google.
La cause probable : une dette technique asymétrique
WordPress 7.0 est conçu pour fonctionner avec PHP 8.3. Si l'environnement serveur n'a pas été mis à niveau en parallèle, le CMS de dernière génération tourne sur un moteur insuffisant — créant exactement ce type d'effondrement de performances.
Les extensions lourdes comme Slider Revolution 6.7.18 (non optimisée pour WordPress 7.0) amplifient ce goulot d'étranglement à chaque chargement de page.
Un prospect qui cherche un bien à louer ou à acheter à Dakar ne patiente pas. Un site qui ne charge pas, c'est un mandat perdu.
