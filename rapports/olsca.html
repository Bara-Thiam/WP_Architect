WordPress 7.0 est à jour, mais Slider Revolution 6.7.16 permet de lire directement vos fichiers de configuration serveur via CVE-2025-9217. WPBakery expose une XSS stockée active. PHP 8.2 atteint sa fin de vie le 31 décembre 2026. Et votre LCP mobile dépasse 22 secondes.
Point 01 — Ce qui fonctionne bien
Un cœur CMS à jour et une infrastructure Cloudflare en place
WordPress Core
7.0
Version majeure stable sortie le 20 mai 2026. Aucune CVE critique connue sur ce cœur. Le moteur du site est propre et à jour.
PHP
8.2.31
Version active avec support sécurité jusqu'au 31 décembre 2026. Pas de problème immédiat — mais une migration vers PHP 8.3+ est à planifier (voir Point 03).
Proxy frontal
Cloudflare
IP d'origine masquée. Protection DDoS active en frontal. La présence de Cloudflare est un atout — à condition que la configuration d'origine soit correctement verrouillée.
Le cœur est sain — le risque vient de la périphérie
WordPress 7.0 déployé dès sa sortie en mai 2026, PHP à jour, Cloudflare en frontal — les décisions d'infrastructure de base sont bonnes. Ce rapport identifie ce que les extensions et la configuration n'ont pas suivi.
Point 02 — Vulnérabilités actives
Slider Revolution et WPBakery — deux CVE actives qui annulent l'effort CMS
Accès initial
Utilisateur authentifié
Un compte avec droits limités — ou un accès obtenu via force brute sur /wp-admin/ exposé en 200.
→
Exploitation
CVE-2025-9217
Slider Revolution 6.7.16 — Path Traversal lors de l'export de médias. Lecture arbitraire de fichiers serveur sans restriction.
→
Résultat
wp-config.php lu
Identifiants complets de la base de données extraits. Accès total aux données clients, formulaires de cotation et contenus éditoriaux.
Slider Revolution 6.7.16 — deux failles cumulées sur la même version
CVE-2025-9217 — Arbitrary File Read (≤ 6.7.36) — Lecture arbitraire de fichiers serveur dont wp-config.php, exposant les identifiants de base de données. Correctif disponible uniquement à partir de la version 6.7.37.
XSS Stockée via import SVG (≤ 6.7.18) — Injection de JavaScript malveillant persistant dans le back-office via l'importation d'un fichier SVG piégé. Permet le vol de sessions administrateur ou la modification de contenu.
WPBakery Page Builder — XSS stockée active (CVE-2025-11161)
La version de WPBakery détectée est inférieure ou égale à 8.6.1, exposant le site à la CVE-2025-11161 (XSS stockée). Résolu uniquement à partir de la version 8.7+. Un attaquant peut injecter du code malveillant persistant dans vos pages de contenu.

Point 03 — Urgence temporelle
PHP 8.2 — fin de support dans 6 mois, accès admin sans restriction
31 déc. 2026
Fin de vie définitive de PHP 8.2. Passé cette date, plus aucun correctif de sécurité ne sera publié pour cette branche. Les nouvelles vulnérabilités découvertes resteront indéfiniment non corrigées sur votre serveur.
Accès wp-admin
200 — Ouvert
/wp-admin/ et /wp-login.php accessibles sans restriction réseau. Conjugué à XML-RPC actif, la surface d'attaque sur les comptes est maximale.
Bypass Cloudflare
Risque latent
Aucune preuve de verrouillage strict côté serveur d'origine (IP Cloudflare only / Authenticated Origin Pulls). Si l'IP réelle est découverte, Cloudflare est contourné.

22,7 secondes de LCP mobile — le moteur de test a signalé une lenteur anormale
Le LCP mobile dépasse 22 secondes — le moteur de test Lighthouse a lui-même émis une alerte signalant que la lenteur du site a perturbé la finalisation des mesures. Ce niveau de lenteur est structurel, pas conjoncturel.
Pour un acteur du transport logistique international, les prospects comparent des prestataires en quelques minutes depuis leur téléphone. Un site à 22 secondes de chargement n'est pas consulté — les demandes de cotation vont directement à un concurrent.
