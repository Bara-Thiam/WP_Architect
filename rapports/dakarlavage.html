Critique
⚠ 3
Tunnel e-commerce exposé — fondations transactionnelles compromises
Votre site affiche des performances solides, mais l'administration ouverte, WooCommerce affecté par une faille CSRF critique (2026) et Slider Revolution cumulant deux CVE actives mettent en danger la confidentialité des données clients et l'intégrité des commandes.
Point 01 — Le risque majeur
Une chaîne de failles qui compromet directement vos commandes
Étape 1
Admin exposé
/wp-admin et /wp-login.php accessibles publiquement (code 200). Cible immédiate pour les attaques brute-force automatisées.
→
Étape 2
Faille WooCommerce
CSRF critique (v5.4–10.5.2) — manipulation d'actions admin à distance via un simple lien piégé envoyé à un gestionnaire connecté.
→
Résultat
Boutique compromise
Création de comptes admin non autorisés, modification des flux de paiement, exfiltration des données et historiques de commandes clients.
WooCommerce 9.6.2 — vulnérabilité CSRF critique documentée en mars 2026
CSRF (Cross-Site Request Forgery) — versions 5.4 à 10.5.2 — Un attaquant peut déclencher des actions administratives à votre insu : création de comptes, modification des flux de paiement, exfiltration de données clients. Version installée : 9.6.2 · Version sécurisée actuelle : 10.8.1.
Pour une boutique qui collecte coordonnées, détails de véhicules et historiques de commandes, une compromission de ce niveau peut engager directement la responsabilité de la structure vis-à-vis de ses clients.
Point 02
Deux CVE actives sur Slider Revolution et protections réseau absentes
WordPress Core
6.7.3
Version 2025 dépassée, antérieure aux correctifs globaux des branches 6.8.x, 6.9.x et 7.0. Mise à jour recommandée.
WooCommerce
9.6.2 CVE
Affecté par la faille CSRF documentée en mars 2026. Version actuelle : 10.8.1. Mise à jour urgente requise.
Slider Revolution
6.7.13 2x CVE
XSS stockée authentifiée + lecture arbitraire de fichiers (CVE-2025-9217). Deux failles cumulées sur cette version.
Slider Revolution 6.7.13 — double vulnérabilité active
XSS Stockée Authentifiée (≤ 6.7.13) — Injection de scripts malveillants persistants dans le back-office, permettant le détournement de sessions de votre équipe de gestion.
CVE-2025-9217 — Arbitrary File Read (≤ 6.7.36) — Un utilisateur avec droits limités peut lire wp-config.php et extraire les identifiants complets de votre base de données.
Protection manquanteNiveauConséquence
X-Frame-Options
Moyen
Votre tunnel d'achat peut être encadré dans une fausse page pour détourner les clics et intercepter les paiements.
Strict-Transport-Security
Moyen
La connexion HTTPS n'est pas imposée — les sessions clients peuvent être rétrogradées vers HTTP non chiffré et interceptées.
X-Content-Type-Options
Mineur
Le navigateur peut interpréter des fichiers malveillants téléchargés comme du code légitime.
